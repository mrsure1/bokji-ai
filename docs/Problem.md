# [프로젝트 기획서] 공공데이터 및 NLP 기반 맞춤형 복지 알리미 개발
## 📌 1. Problem (문제 정의) 및 Solution (해결 방안)

본 프로젝트는 대한민국 정부가 제공하는 복지 혜택의 접근성 문제를 최신 인공지능(AI) 및 자연어 처리(NLP) 기술로 해결하기 위해 아래와 같이 문제를 정의하고 구체적인 기술적 해결 방안을 수립합니다.

---

### ❌ 문제 1: '신청주의' 행정으로 인한 복지 사각지대 발생
* **현상의 문제점:** 대한민국의 복지 제도는 개인이 직접 정보를 알아보고 신청해야만 지급하는 '신청주의'를 채택하고 있습니다. 정부24의 '보조금24'가 존재하지만, 사용자가 직접 로그인 후 수동으로 '조회'해야만 내역을 보여주므로 바쁜 취약계층은 혜택을 놓치기 쉽습니다.
* **💡 AI 기반 해결 방안:** **선제적 데이터 매칭 및 자동 푸시(Push) 시스템**
    * 사용자가 앱 가입 시 입력한 기본 정보(연령, 거주지, 가구 특성 등)를 바탕으로, 조건에 맞는 새로운 정부 공고가 등록되면 사용자가 앱을 켜지 않아도 AI가 먼저 스마트폰 알림으로 혜택을 배달합니다.

### ❌ 문제 2: 난해한 행정·법률 용어로 인한 '정보 해석'의 장벽
* **현상의 문제점:** 공공데이터포털 API로 가져오는 복지 공고문 원본은 법률적 기준과 행정 전문 용어가 줄글로 복잡하게 얽혀 있어, 비전공자나 시니어 계층은 핵심 내용(지원 금액, 신청 방법)을 파악하기 어려워 신청을 포기합니다.
* **💡 AI 기반 해결 방안:** **LLM 기반 행정 문서 자동 요약 (Text Summarization)**
    * 최신 대형 언어 모델(LLM)을 활용하여 복잡한 행정 줄글을 읽고, "1줄 요약", "내가 받을 혜택", "준비할 서류" 형태로 초등학생도 이해할 수 있는 쉬운 일상어로 자동 요약하여 UI에 시각화합니다.

### ❌ 문제 3: 지자체별(시·군·구) 고유 복지 혜택의 파편화
* **현상의 문제점:** 중앙정부 혜택 외에도 각 지방자치단체가 자체 집행하는 청년수당, 교통비 지원 등이 매우 많으나, 이 데이터들이 '정부24'와 '복지로'에 산재해 있어 사용자가 자신의 거주지 기준 혜택을 통합해서 보기 어렵습니다.
* **💡 AI 기반 해결 방안:** **이종(異種) 공공데이터 API 통합 파이프라인 구축**
    * 행정안전부(보조금24) API와 한국사회보장정보원(복지로 지자체 서비스) API를 동시에 연동하는 백엔드 수집기를 구축하여, 사용자의 거주지(시·군·구)에 100% 매칭되는 특화 복지 데이터를 한곳에 통합합니다.

### ❌ 문제 4: 일상적 맥락(자연어)을 인지하지 못하는 기존 키워드 검색의 한계
* **현상의 문제점:** 기존 공공기관 사이트는 정확한 단어가 일치해야만 결과가 나오는 '키워드 매칭' 방식입니다. "갑자기 퇴사해서 방세 낼 돈이 없다"와 같이 자신의 힘든 상황을 일상적인 대화체로 질문하면 기존 시스템은 아무런 혜택도 추천하지 못합니다.
* **💡 AI 기반 해결 방안:** **RAG(검색 증강 생성) 기반 문맥·의미 검색 구현**
    * 공고문 텍스트를 인공지능이 이해할 수 있는 수학적 벡터값으로 변환(Embedding)하여 저장합니다. 사용자가 하소연하듯 질문해도 AI가 문맥의 '의미'를 파악해 가장 연관성이 높은 실업급여나 긴급복지지원 공고를 찾아냅니다.

### ❌ 문제 5: 실시간 API 호출로 인한 시스템 부하 및 속도 저하 (기술적 제약)
* **현상의 문제점:** 유저가 검색할 때마다 실시간으로 정부 서버 API를 호출하여 수천 개의 텍스트를 LLM에게 읽히면 답변 속도가 수십 초 이상 지연되며, 정부 공공데이터 서버의 일일 호출 트래픽 제한을 초과해 서비스가 마비될 수 있습니다.
* **💡 AI 기반 해결 방안:** **주기적 로컬 스크래핑 및 Vector DB 기반 하이브리드 아키텍처**
    * 매일 새벽 사용량이 적은 시간에 정부 API를 호출하여 최신 데이터를 내 서버의 관계형 DB(RDB)와 벡터 데이터베이스(Vector DB)에 미리 동기화해 둡니다. 이를 통해 유저 검색 시 1~2초 내에 안전하고 초고속으로 RAG 답변을 생성합니다.
